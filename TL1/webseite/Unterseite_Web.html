Grundprinzip des Webs

Auf der Client-Side wird eine Anfrage durch einen Nutzer (durch Nutzung eines Browsers) oder ein Protokoll, etc. ausgelöst,
worauf die angefragten Ressourcen vom Server geholt werden und zurückgebracht werden,
um zum Beispiel vom Browser für den Nutzer zu Anzeige gebracht werden (Rendering).

Server-Client-Prinzip

Client und Server sind Programme bzw. Prozesse. Der Server wartet stehts auf Anfragen des Clients und lädt HTML-Dokumente (auch Lieferung von Ressourcen genannt).
Die Anfragen werden immer vom Client initialisiert (Client Pull Prinzip), wobei der Browser auf der Clientseite verschiedene Rollen einnehmen kann.

IP Protokoll

Das IP Protokol verpackt zu übertragenden Daten in Pakete (paketorientiert) und leitet sie an andere Rechner mithilfe von IP-Adresse weiter.
Da Sender und Empfänger sich nicht kennen ist es verbindungslos und damit auch unzuverlässig.
Es gibt dabei zwei Arten von IP-Adressen (IPv4 und IPv6). IPv4 besteht aus vier Blöcken, die jeweils die Werte von 0 bis 255 annehmen können,
während IPv6 sich in acht mal 16 Bit unterteilt und über verschiedene Sicherheitsfeatures verfügt.
Eine besondere Adressen im IPv4 Format ist zum Beispiel 127.0.0.1 für den Localhost.

TCP Protokoll

Das TCP Protokoll baut auf dem IP Protokoll auf.
Es ist für das konkrete Zusammensetzen beim Adressaten verantwortlich
und löst das Problem der Unzuverlässigkeit des IP Protokolls durch Retransmisson (erneute Zusendung) verlorengegangener Pakete.
Auch wird beim TCP Protokoll das Konzept der Ports eingeführt (d.h. kann IP-Adresse Portnummer beifügen, denn IP Protokoll kann (nur) Rechner als Ganzes adressieren)

Port

Ports sind Verbindungspunkte auf Rechnern, die Programmen und Prozessen auf dem Zielrechner erlauben über einen gewissen Port zu kommunizieren. Sie sind also Verbindungspunkte zu Programmen am Zielrechner.

Domain Name Service (DNS)

Der Domain Name Service ist ein verteilter hirarchischer Verzeichnisdienst, der den Namnesraum des Internets verwaltet.
Er ist in Zonen unterteilt, für die unabhänige Administratoren zuständig sind.
Seine Aufgabe besteht vor allem darin Domainnamnen in IP Adressen zu übersetzen (forward lookup).

Hypertext Transfer Protokoll

Das HTTP Protokoll ist ein Anwendungsprotokoll des World Wide Webs auf Basis des TCP und des IP-Protokolls.
Es ist also die gemeinsame Sprache von Web-Client und Web-Server und beschäftigt sich damit, was logisch geschieht und welche Befehle genutzt werden.
Physikalisch funktioniert es durch TCP und IP gemäß den Schichten des OSI-Schichten-Modells und sorgt für Transfer (angefordert, geliefert) von Ressourcen/Daten beliebiger Formate zwischen Server und Client.
Es ist ein zustandsloses Protokoll, da kein Verbindungszusammenhang zwischen Client und Server aufgebaut wird, d.h. Verbindung wird bei jedem neuen Request neu aufgebaut (auch z.B. bei Navigieren auf Webseiten).
Das HTTP Protokoll basiert auf dem Request Response Paradigma, d.h. der Server schickt nur eine Response, wenn der Client vorher ein Request gesendet hat.

Ressourcen

Ressourcen sind vom Client bei der Anfrage spezifizierte Entitäten (häufig HTML-Dokumente und Bilder)
Beispiele: Video-Sequenzen; interaktive 3D-Medien; PDF-Dokumente; ZIP-Dateien; serverseitige Programme ((z.B. in PHP): werden nicht an den Client geliefert, sondern auf Server-Seite ausgeführt -> liefern als Resultat typischerweise HTML-Dokumente).

Uniform Resource Identifier (URI)

Die URI ist eine Zeichenabfolge zur Identifizierung einer abstrakten oder physischen Ressource.
Sie ist ein sehr allgemeines Konzept und findet auch im sogennanten Semantic Web anwendung.

URL und URN

URL und URN sind Unterarten von URIs.
URL steht für Uniform Resource Locator und ) identifiziert eine Ressource über ihren primären Zugriffsmechanismus (d.h. Ort der Ressource im Netz, wo sie abrufbar ist)
URN steht für Uniform Resource Name und identifiziert mit dem urn Schema eine Ressource mittels eines vorhandenen oder frei zu vergebenden Namens (z.B. ISBN Nummer).

IP Adressformate

zwei Versionen: IPv4 und IPv6 (größerer Adressraum, Sicherheitsfeatures (z.B. IPSec), …)
IPv4 besteht aus 4 Blöcken mit je Zahlen zwischen 0 und 255 (8 Bit)
Somit ist N = 232 Adressen, wovon einige Sonderadressen sind. (z.B. 127.0.0.1 für localhost).

Wichtige Ports

Port 80: Wellknown Port vor allem für HTTP Requests
Port 25: für Mailserver
Port 53: für DNS Server

wichtige HTTP Methoden

GET: Ressource wird unter Angabe eines URI, in der Parameter kodiert werden, vom Server angefordert
POST: schickt unbegrenzte Menge an Daten zur Verarbeitung zum Server, wobei die Daten werden als Inhalt der Nachricht übertragen werden.
