HTML Dokumente

Ein HTML Dokument besteht immer aus einem head-Teil und einem body-Teil, indem Inhalte zwischen Tags stehen.
Somit sind die Minimaleinheiten eines HTML Dokuments der html Tag, der das Wurzelelement ist, der head Tag, der Titel, Metadaten, style und script tags beinhaltet und der body Tag, zwischen dem der für den Browsernutzer sichbare Inhalt sich befindet.
HTML Dokumente sind versionslos und damit sowohl abwärtskompatibel, wie auch für künftige Versionen nutzbar.

Block- und Zeichenelemente

BLOCKELEMENTE:
bescheiben, die Struktur eines Dokuments. In einem einfachen HTML Dokument werden die Blöcke nacheinander angezeigt.

ZEICHENELEMENTE:
werden innerhalb eines Blocks genutzt und dienen der Formatierung. Dabei kann zwischen logischer und absoluter Formatierung unterschieden.

Strukturierung von Webseiten

Webseiten werden durch div-Elemente strukturiert. In HTML5 erlauben die div-Elemenete als spezifische Tags auch Aussagen über den Inhalt, wie z.B. header, footer, nav, etc.
Sie machen nicht nur die Struktur der Webseite deutlich, sondern helfen auch dem Browser die Inhalte von spezifischen Elementen zu verstehen.
Die Positionierung der div-Elemente wird durch CSS definiert.

Formulare

Formulare können vom Nutzer ausgefüllt werden und dann über einen Submit-Button zum Server übertragen werden.
Die Übertragung zum Server erfolgt mittels HTTP und meist der Methode POST. Der beim Submit-Button hinterlegte PHP-Code wird dann auf dem Server ausgeführt und die Ergebnisse, die vom Server z.B. in ein HTML Template eingefügt werden können, mittels eines Array, werden zum Client zurückübertragen.

Offline-Anwendungen

Im Zuge der Konkurrenz zwischen narrativen Apps und HTML bei mobilien Endgeräten wurde für HTML eine Funktionalität von Anwendungen trotz Verbindungsabbruch ermöglicht.
Hierbei wird zunächst ein Cache-Manifest angelegt, in dem festgelegt wird, was gespeichert wird und was nicht und was im Fehlerfall angezeigt wird.
Dann werden die htaccess files angepasst und das Cache-Manifest verlinkt.

CSS Einbindung

Es gibt 3 Möglichkeiten CSS einzubinden.
Man kann CSS in das Element einbinden, was aber den Nachteil hat, dass es schlecht wartbar ist, zu Redundanzen führen kann und keine klare Trennung vom Inhalt hat.
Man kann CSS auch im head einbinden, was zwar eine Trennung vom Inhalt ermöglicht, jedoch nicht über mehrere Seiten hinweg genutzt werden kann.
Die letzte Möglichkeit ist ein extra CSS Stylesheet zu erzeugen, das eine vollständige Trennung von Inhalt und Präsentation erlaubt, über mehrere Seiten wiederverwendbar ist und einfach zu warten ist. Außerdem erlaubt es eine Trennung nach Funktionalitäten.

Document Object Model

Das DOM ist auf der einen Seite ein Modell und auf der anderen Seite auch eine Programmierschnittstelle.
Es ist ein plattform- und sprachneutrales Interface und ermöglicht eine dynamischen Zugriff und Veränderung des Inhalts, der Struktur und Gestalt von Dokumenten, sowie auch die Weiterverarbeitung und Integration der Ergebnisse in das Dokument.

Boxmodell

definiert die Berechnung der Breite und Höhe von Elementen
- Gesamtbreite eines Elements errechnet aus einer Addition
- der Breite des Elementinhalts (width)
- des Innenabstands (padding)
- der Rahmenstärke (border-width) und
- des Außenabstands (margin)
- gilt analog für die Höhe (height)

Pseudoklassen und Pseudoelemente

PSEUDOKLASSEN:
Es werden spezielle Zustände von Elementen adressiert.
Beispiel: Zeige an, ob ein Link schon einmal angeklickt wurde oder nicht

PSEUDOELEMENTE:
werden verwendet, um Teile eines Elements anzugeben.
Beispiel: Formatiere die erste Zeile oder den ersten Buchstaben in einer anderen Farbe.

Kaskardierung

CSS steht für Cascading Stylesheets, was impliziert, dass mehrere Stylesheets angewendet werden können.
Es werden also mehrere Stylebeschreibungen zusammengefasst und auf ein Dokument angewendet.
Dabei werden externe, interne und durch den Nutzer in seinem Browser registrierte Sytlesheets unterschieden.
Treten für die Stylebschreibung eines Elements Konflikte auf, so werden sie durch komplexe Vorrangsregeln aufgelöst. So überschreiben zum Beispiel neuere Stylesheets ältere Stylesheets, etc.

Escape Sequenzen

... werden benötigt, wenn Zeichen verwendet werden, die in der Markup Language oder Programmiersprache selbst eine Bedeutung haben, wie <
oder auch Sonderzeichen, wie ü, die sonst Mojibake verursachen würden.

Farbangaben

Über vordefinierte Farbnamen oder Hexadezimalzahlen (Groß- und Kleinschreibung nicht wichtig) oder...
- rgb (...)
- rgba (...)
- hsl (...)

Cache-Manifest

CACHE MANIFEST:
wird im Appcache gespeichert.

NETWORK:
wird nicht im Appcache gespeichert

FALLBACK:
wird im Fehlerfall angezeigt

Selektoren

Selektoren wählen mit Hilfe des DOMs Elemente aus und weisen ihnen Werte und Attribute zu (z.B. Schriftart, Schriftfarbe, etc.).
Es kann über Eltern-Kind-Beziehung, über ID und Attribut-Selektoren, selbstdefinierte Klassen und Pseudoklassen und -elemente selektiert werden.
